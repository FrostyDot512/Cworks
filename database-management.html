// how it works

Our Process

A structured workflow from requirements to a production-ready database.

01

Requirements & Data Audit

We start by understanding your business, what data you're storing or need to store, and how it's being used. If you already have a database or spreadsheets, we audit what you have and identify what needs to be preserved, restructured, or discarded.

02

Schema Design

We design the database structure — tables, columns, data types, primary and foreign keys, and relationships. We produce an entity-relationship (ER) diagram and share it with you for review before any database is created.

03

Build & Configuration

We create the database, set up indexes, configure user permissions, and write any stored procedures or triggers your application requires. We test the structure thoroughly before populating it with real data.

04

Data Migration & Population

We clean and import your existing data into the new structure — whether it's coming from spreadsheets, a legacy system, or another database. We verify record counts and data integrity once the migration is complete.

05

Testing & Optimisation

We run the queries your application will use in production, profile performance, and make optimisations. Slow queries get rewritten or indexed. We don't sign off until everything runs efficiently under realistic load.

06

Handover & Ongoing Support

We deliver the complete schema documentation, backup configuration, and access credentials. We walk you through the structure and stay available post-handover for questions. Ongoing management and administration can be arranged as a retainer.

// faq

What database technology do you use?

We primarily work with MySQL, which is the most widely used relational database system and integrates cleanly with PHP and Python. We can also work with SQLite for lightweight applications or PostgreSQL if your stack requires it.

Can you migrate data from our existing system?

Yes. We handle migrations from spreadsheets, legacy databases, CSV exports, and other MySQL instances. We clean the data during migration and verify it's all landed correctly before switching over.

Will the database work with our existing application?

If your application connects to MySQL via PHP, Python, or any standard MySQL connector, yes. We'll coordinate with your development team or — if Cworks is also building the application — handle the integration ourselves.

How is the data backed up?

We configure automated daily backups stored either on the same server or an external location. We'll walk you through the backup schedule and how to restore from a backup on handover.

Can you take over management of an existing database?

Yes. If you have an existing database that needs optimisation, restructuring, or ongoing management, share the details and we'll audit it and quote for the work needed.
